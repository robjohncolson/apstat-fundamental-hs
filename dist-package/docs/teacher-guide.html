AP Statistics PoK Blockchain

Implementing blockchain-based peer learning in your AP Statistics classroom

üéØ Educational Philosophy

This system implements constructivist learning through blockchain technology. Instead of providing authoritative answers, students build knowledge collaboratively through peer consensus and statistical validation.

üîë Key Pedagogical Benefits

  • Active Learning: Students must explain their reasoning, not just provide answers
  • Peer Assessment: Students learn by evaluating classmates' work
  • Metacognition: Confidence ratings encourage self-reflection
  • Statistical Thinking: Students experience convergence, distributions, and consensus in real-time
  • Technology Integration: Authentic use of blockchain concepts in mathematics

üöÄ Classroom Implementation

Pre-Class Setup (15 minutes)

  1. Download and extract the ZIP file to a shared network location
  2. Test the system on one computer to ensure it loads properly
  3. Prepare backup USB drives with the system files
  4. Review the day's questions in the curriculum.json file

Student Distribution Options

Option 1: Network Share
  • Place the dist-package folder on a shared network drive
  • Students navigate to \\server\apstat\index.html
  • Each student's progress saves locally to their browser
  • No internet required once files are accessed
Option 2: USB Distribution
  • Copy the system to USB drives or SD cards
  • Students copy to their local machines
  • Fully offline - works without network at all
  • Each student has independent instance
Option 3: Individual Downloads
  • Email the ZIP file to students
  • Students extract and run locally
  • Great for homework or remote learning
  • Each student manages their own copy

üìä Assessment and Grading

Using Reputation Scores

Student reputation scores can inform (but not replace) traditional assessment:

Formative Assessment
  • Monitor daily participation through event counts
  • Identify struggling students through low confidence patterns
  • Use convergence rates to gauge class understanding
  • Review outlier flags to catch misconceptions early
Summative Assessment Integration
  • Reputation scores as participation/engagement grade (10-20%)
  • FRQ peer-scoring accuracy as assessment skill grade
  • Confidence calibration as metacognitive awareness score
  • Traditional AP-style assessments remain primary (60-70%)

üéì Facilitating Learning

Day 1: Introduction Session

  1. Demo (10 min): Show the tutorial on the projector
  2. Guided Practice (15 min): Have students work through first attestation together
  3. Explanation (10 min): Discuss how consensus relates to statistical inference
  4. Independent Work (15 min): Students complete 2-3 additional questions

Ongoing Sessions

üîç Monitoring Tools
  • Walk around: Check student screens for confidence patterns
  • Convergence watching: Project class consensus rates on board
  • Question selection: Note which questions cause low convergence
  • Help requests: Students struggling with invariants need conceptual support

Discussion Prompts

  • "Why did this question have low convergence? What does that tell us?"
  • "How is peer consensus similar to scientific peer review?"
  • "When might statistical consensus be wrong? How do we guard against it?"
  • "How does your confidence affect your learning strategy?"

üîß Technical Management

System Requirements

  • Browsers: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
  • Memory: 512MB RAM minimum, 1GB recommended
  • Storage: 50MB per student for local data
  • Network: None required after initial file distribution

Troubleshooting Common Issues

Students Can't Load System
  • Check browser version (update if necessary)
  • Verify JavaScript is enabled
  • Clear browser cache and cookies
  • Try private/incognito mode
Performance Issues
  • Close other browser tabs
  • Check available RAM (close other applications)
  • Use Chrome for best performance
  • Disable browser extensions if necessary

Backing Up Student Progress

  1. Remind students to use "Export State" regularly
  2. Create a shared folder for student exports
  3. Consider periodic "save parties" where everyone exports
  4. Keep backup copies of the system files

Differentiated Instruction

For Advanced Students
  • Challenge them to achieve high reputation with high confidence
  • Have them mentor struggling classmates
  • Assign investigation of the 13 invariants
  • Create new questions for peer testing
For Struggling Students
  • Focus on building confidence through low-stakes questions
  • Pair with stronger students for FRQ scoring practice
  • Use tutorial mode repeatedly
  • Emphasize learning over reputation building

‚ö†Ô∏è Important Reminders

Academic Integrity

  • Emphasize that the goal is learning, not gaming the reputation system
  • Monitor for unusual consensus patterns that might indicate coordination
  • Discuss honest confidence self-assessment
  • Address any attempts to manipulate the system

Learning vs. Gaming

  • Some students may try to optimize reputation over learning
  • Emphasize process over outcome in your grading
  • Use reflection questions to assess true understanding
  • Celebrate thoughtful minority positions, not just high scores
